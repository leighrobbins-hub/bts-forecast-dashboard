# Back-to-School Forecast Dashboard

Automated weekly analysis of BTS forecast vs. run rate capacity with supply vs. utilization problem classification.

**Live Dashboard:** https://YOUR-USERNAME.github.io/bts-forecast-dashboard/

---

## Features

- ✅ **Weekly automated updates** (runs every Monday for WBR prep)
- ✅ **Looker API integration** for run rate and utilization data
- ✅ **Supply vs. Utilization classification** (extends LSAT investigation to all subjects)
- ✅ **Interactive filtering** (problem type, search)
- ✅ **Sortable columns** (click headers to sort)
- ✅ **Color-coded problems** (red = supply, orange = utilization, green = on track)

## Quick Start

### 1. Create GitHub Repository

```bash
# Create new repo on GitHub (call it bts-forecast-dashboard)
# Then clone locally:
git clone https://github.com/YOUR-USERNAME/bts-forecast-dashboard.git
cd bts-forecast-dashboard

# Copy all files from this folder to your repo
```

### 2. Set Up GitHub Secrets

Go to your repo → Settings → Secrets and variables → Actions → New repository secret

Add these secrets:
- `LOOKER_CLIENT_ID`: Your Looker API client ID
- `LOOKER_CLIENT_SECRET`: Your Looker API client secret
- `LOOKER_API_URL`: https://varsitytutors.looker.com (or your Looker URL)

**How to get Looker API credentials:**
1. Log into Looker
2. Go to Admin → Users → Your User → Edit
3. Under "API Keys" → Create new API3 Key
4. Copy the Client ID and Secret

### 3. Update Looker Query IDs

Edit `scripts/fetch_looker_data.py` and update these query IDs:

```python
# Line 55: Run rate query ID
QUERY_ID = "YOUR_RUN_RATE_QUERY_ID"  

# Line 72: Utilization query ID  
QUERY_ID = "YOUR_UTILIZATION_QUERY_ID"
```

**How to get query IDs:**
1. Open the Looker look/query you want to use
2. Look at the URL: `https://varsitytutors.looker.com/looks/XXXXX` or `/explore/...?qid=XXXXX`
3. The query ID is the XXXXX part

**Queries you need:**
- **Run Rate Query:** Should return subject-by-month contracting data (like your tutoring_tutors file)
- **Utilization Query:** Should return tutors with assignments within 30 days (like your second file)

### 4. Enable GitHub Pages

1. Go to repo → Settings → Pages
2. Source: Deploy from a branch
3. Branch: `gh-pages` (will be created by Action)
4. Save

### 5. Upload Initial Data

Create `data/` folder and add:
- `monitoring_table.xlsx` - Pierre's forecast output (manual upload weekly)
- `run_rates.csv` - If Looker API doesn't work, upload manually
- `utilization.csv` - If Looker API doesn't work, upload manually

### 6. Run Initial Workflow

```bash
# Push everything to GitHub
git add .
git commit -m "Initial setup"
git push origin main

# The GitHub Action will run automatically
# Or trigger manually: Go to Actions tab → Update BTS Forecast Dashboard → Run workflow
```

### 7. Access Your Dashboard

After the Action completes (2-3 minutes):
- Go to https://YOUR-USERNAME.github.io/bts-forecast-dashboard/
- You should see the live dashboard!

## How It Works

### Weekly Automation (Every Monday 6 AM UTC)

1. **GitHub Action triggers** → runs workflow
2. **Fetches Looker data** → run rates + utilization (via API)
3. **Reads Pierre's forecast** → from data/monitoring_table.xlsx (manual upload)
4. **Runs analysis** → smoothing + problem classification
5. **Generates dashboard data** → data.json
6. **Deploys to GitHub Pages** → live dashboard updates

### Manual Updates (When Needed)

**Upload Pierre's latest forecast:**
```bash
# Copy his monitoring_table.xlsx to data/ folder
# Commit and push:
git add data/monitoring_table.xlsx
git commit -m "Update forecast - Week XX"
git push

# Action runs automatically, dashboard updates in 2-3 minutes
```

**Override Looker data manually:**
```bash
# If API isn't working, upload CSVs manually:
git add data/run_rates.csv data/utilization.csv
git commit -m "Manual data update"
git push
```

## Troubleshooting

### Dashboard shows "Error loading data"
- Check that `dashboard/data.json` was generated
- Run `python scripts/run_analysis.py` locally to test
- Check GitHub Actions logs for errors

### Looker API not working
- Verify secrets are set correctly in GitHub
- Check query IDs in `fetch_looker_data.py`
- Fallback: Upload CSVs manually to `data/` folder

### GitHub Pages not deploying
- Check Settings → Pages is enabled
- Verify Action completed successfully (green checkmark)
- Wait 2-3 minutes for GitHub Pages to rebuild

### Data looks wrong
- Verify `monitoring_table.xlsx` is latest from Pierre
- Check `data/analysis_results.csv` to see intermediate results
- Re-run Action manually to force refresh

## Customization

### Change update schedule
Edit `.github/workflows/update-dashboard.yml`:
```yaml
schedule:
  - cron: '0 6 * * 1'  # Monday 6 AM UTC
  # Change to: '0 14 * * 5' for Friday 2 PM UTC, etc.
```

### Add more metrics to dashboard
1. Update `scripts/run_analysis.py` to calculate new metrics
2. Add columns to `dashboard/index.html` table
3. Push changes

### Change color coding
Edit CSS in `dashboard/index.html` (lines 30-60)
